# Voice policy (template)

A second brain that recalls facts but cannot sound like you will draft content
you would never sign. This file is the quality gate that keeps first-person
output in the person's own voice. Replace the contents with your own; keep the
shape.

> This is a **generic template**. The real version describes a specific person.

## The voice in one paragraph

> The single anchor every draft is measured against. Example placeholder:
> "Writes plainly and specifically. Explains *why* before *how*. Dry humour,
> never hype. Would rather be precise than impressive."

## What the voice IS

- Specific — names real things, uses concrete examples over abstractions.
- Reasoned — shows the trade-off, not just the conclusion.
- Honest about limits — says what a thing does *not* do.

## What the voice IS NOT

- Not breathless. No "thrilled to share", no rocket-ship enthusiasm.
- Not corporate. No mission-statement filler.
- Not vague. No claim without a concrete anchor behind it.

## Forbidden phrases

> The exact strings that must never appear regardless of channel. Example
> placeholders: "excited to share", "hot take:", "game-changer", validation-bait
> questions.

## The screenshot test

Before publishing first-person content, check it against a few gates:

1. Could a generic assistant have written this? If yes, rewrite.
2. Is there a specific, named thing in it? If no, add one.
3. Does it claim something it cannot back? If yes, cut the claim.

A draft that fails two or more gates is rewritten, not shipped.

## How agents use it

Any agent producing first-person content for this person must load this policy,
draft against the one-paragraph anchor, and run the screenshot test before
returning a draft. Voice compliance is a quality gate, not a stylistic
preference.
